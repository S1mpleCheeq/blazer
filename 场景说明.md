# 一、场景（升级版）⚡🐕🚁

## 场景名称

**变电站巡检中的突发告警插队处理**

## 场景设定

当前系统正在执行一个**常规巡检任务（Task A）**：

* 巡检范围：1号主变区 → GIS区 → 控制楼
* 当前执行进度：

  * 已完成：1号主变区巡检
  * 正在执行：GIS区巡检

此时，系统接收到一个**突发任务（Task B）**：

* 2号主变区出现**高温异常告警（高优先级）**

---

## 演示目标（核心卖点）✨

你要让客户看到这三件事：

### 1️⃣ 当前任务不会丢

* 系统**记录 Task A 的执行状态**
* 包括：

  * 已完成节点
  * 当前执行位置
  * 未完成节点

👉 对应你们的 **Memory / 工作记忆能力**

---

### 2️⃣ 高优任务可以“插队执行”

* Host Agent 判断 Task B 优先级更高
* **中断 Task A（挂起）**
* 单独拆 Task B 的 DAG 并执行

👉 对应 **任务编排 + 调度能力**

---

### 3️⃣ 执行完自动恢复

* Task B 完成后
* 自动恢复 Task A 的剩余 DAG
* 从**中断点继续执行**

👉 这是最关键的 demo 亮点 🔥

---

# 二、演示用 Prompt（这次是“剧情式”）🎬

这次不是多个独立 DAG，而是**一个连续剧情**。

---

## Prompt 1：启动常规巡检（Task A）

```text
请对变电站执行一次例行巡检：完成1号主变区、GIS区和控制楼巡检，并输出巡检结果。
```

👉 演示点：

* Host 拆 DAG（顺序结构）
* 开始执行
* Memory 记录：

  * Task A
  * 当前节点：GIS区巡检

---

## Prompt 2：注入突发任务（关键）

```text
突发告警：2号主变区检测到高温异常，请立即进行复核，优先级高于当前任务。
```

👉 系统行为（重点展示）：

### Host Agent 输出：

```text
- 检测到高优任务 Task B
- 当前 Task A 状态：已完成 N1，正在执行 N2
- 挂起 Task A
- 创建 Task B DAG
- 切换执行 Task B
```

👉 这一步是核心卖点 💥
